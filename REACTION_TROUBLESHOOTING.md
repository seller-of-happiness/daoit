# Устранение проблем с синхронизацией реакций в чате

## Проблема
Реакции в чате не появляются у других пользователей в режиме реального времени через WebSocket.

## Диагностика проблемы

### 1. Запуск диагностического скрипта
Скопируйте и выполните содержимое файла `debug-reactions.js` в консоли браузера:

```javascript
// Вставьте код из debug-reactions.js
```

Скрипт проверит:
- Состояние WebSocket соединения (Centrifuge)
- Активность чата и сообщений
- Аутентификацию пользователя
- Настроит мониторинг событий

### 2. Возможные причины проблем

#### 2.1 WebSocket соединение не установлено
**Симптомы:**
- `checkCentrifugeConnection()` показывает `connected: false`
- В консоли ошибки подключения к WebSocket

**Решение:**
```javascript
// Переподключение к WebSocket
const centrifugeStore = window.$pinia.state.value.centrifuge
await centrifugeStore.initCentrifuge()
```

#### 2.2 Пользователь не подписан на канал
**Симптомы:**
- WebSocket подключен, но события не приходят
- Пустой массив подписок в диагностике

**Решение:**
```javascript
// Проверить подписки
const chatStore = window.$pinia.state.value.chatStore
chatStore.subscribeToUserChannel()
```

#### 2.3 Некорректный формат данных событий
**Симптомы:**
- События приходят, но не обрабатываются
- В консоли предупреждения о некорректных данных

**Решение:**
Проверить формат событий в консоли и сравнить с ожидаемым:
```javascript
// Ожидаемый формат события реакции
{
  event_type: "reaction_added" | "reaction_removed" | "new_reaction",
  data: {
    chat_id: number,
    message_id: number,
    reaction_type_id: number,
    user_id: string
  }
}
```

## Внесенные исправления

### 1. Улучшена обработка оптимистичных реакций
- Добавлена задержка в 500ms для синхронизации с WebSocket
- Исправлена логика очистки оптимистичных реакций
- Улучшена обработка эксклюзивности реакций

### 2. Усилена надежность локального обновления
- Использование глубокого копирования для избежания мутаций
- Более строгая проверка пользователей при фильтрации реакций
- Использование `splice()` для гарантированного обновления реактивности

### 3. Расширено логирование для отладки
- Детальное логирование событий реакций в Centrifuge
- Цветные эмодзи для различения типов событий
- Полная информация о данных событий

### 4. Оптимизирована обработка WebSocket событий
- Убрана избыточная проверка применения изменений
- Упрощена логика обновления реактивности
- Улучшена обработка ошибок

## Тестирование исправлений

### 1. Автоматическое тестирование
```javascript
// Запустить после диагностики
testReaction()
```

### 2. Ручное тестирование
1. Откройте чат в двух разных браузерах/вкладках под разными пользователями
2. Добавьте реакцию в одном браузере
3. Проверьте появление реакции во втором браузере без обновления страницы
4. Измените/удалите реакцию и проверьте синхронизацию

### 3. Проверка логов
В консоли должны появляться следующие сообщения:
```
🎭 Centrifuge reaction event received: {...}
🎭 Обработка реакции: {...}
➕ Добавлена эксклюзивная реакция локально: {...}
✅ Принудительное обновление реактивности сообщений
```

## Мониторинг в продакшене

### Добавление метрик
Рекомендуется добавить метрики для отслеживания:
- Количество WebSocket переподключений
- Время обработки событий реакций

### Алерты
Настроить алерты на:
- Частые разрывы WebSocket соединения
- Высокий процент ошибок обработки реакций
- Долгое время отклика событий
