# Инструкции по тестированию обновления реакций через WebSocket

## Быстрый тест

1. Откройте приложение и перейдите в чат
2. Откройте консоль браузера (F12)
3. Вставьте и выполните содержимое файла `debug-centrifuge-fix.js`
4. Наблюдайте за логами в консоли

## Ручное тестирование

После выполнения скрипта используйте функции из `window.debugCentrifuge`:

```javascript
// Тестирование добавления реакции
window.debugCentrifuge.testReaction(16, 338, 3, "test-user-123", "new_reaction");

// Тестирование удаления реакции
window.debugCentrifuge.testReaction(16, 338, 3, "test-user-123", "reaction_removed");

// Тестирование с реальными данными из вашего примера
window.debugCentrifuge.testWithData({
  event_type: "new_reaction",
  data: {
    chat_id: 16,
    message_id: 338,
    reaction_type_id: 3,
    user_id: "61fd72aa-9457-42b8-9098-7c0a676cad6e"
  }
});
```

## Что проверять

1. **Локальное обновление**: Реакции должны появляться/исчезать без перезагрузки страницы
2. **Логирование**: В консоли должны появляться сообщения:
   - `🎭 Обработка реакции:`
   - `🎭 Данные реакции:`
   - `➕ Добавлена реакция локально:` или `➖ Удалена реакция локально:`
   - `✅ Реакция обновлена локально для сообщения:`

3. **Fallback**: При ошибках должно происходить обновление через `fetchMessages`

## Ожидаемый результат

- Реакции обновляются мгновенно
- Интерфейс остается отзывчивым
- Нет дублирования реакций
- Корректное отображение количества реакций и пользователей
