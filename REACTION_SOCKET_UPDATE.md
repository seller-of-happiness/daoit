# Обновление реакций через WebSocket

## Описание изменений

Добавлена поддержка обновления реакций в реальном времени через WebSocket соединение с использованием Centrifuge.

## Структура данных

Сокет получает данные в следующем формате:

```javascript
{
  "channel": "chats:user#61fd72aa-9457-42b8-9098-7c0a676cad6e",
  "pub": {
    "data": {
      "event_type": "new_reaction",
      "data": {
        "chat_id": 16,
        "message_id": 338,
        "reaction_type_id": 3,
        "user_id": "user-uuid-here"
      }
    }
  }
}
```

## Поддерживаемые события

- `new_reaction` - добавление новой реакции
- `reaction_removed` - удаление реакции
- `reaction_added` - альтернативное название для добавления реакции

## Внесенные изменения

### 1. chatStore.ts

#### Обновлен метод `handleReactionUpdate`:
- Добавлено извлечение `user_id` из данных
- Добавлен вызов локального обновления реакций
- Добавлен fallback на перезагрузку сообщений при ошибке

#### Добавлен метод `updateMessageReactionLocally`:
- Локальное добавление/удаление реакций без перезагрузки всех сообщений
- Поддержка различных форматов данных реакций
- Обработка ошибок с fallback на полную перезагрузку

### 2. useReactions.ts

#### Улучшена обработка различных форматов данных:
- Добавлена поддержка формата `{user_id: "...", reaction_type_id: 2}`
- Добавлена поддержка формата `{user: "...", reaction_type: 2}`
- Улучшено извлечение информации о пользователях из участников чата

#### Обновлена функция `extractMyReactionFromServer`:
- Добавлена поддержка новых форматов данных
- Улучшена обработка различных полей реакций

### 3. debug-centrifuge-fix.js

#### Добавлены функции тестирования реакций:
- `testReactionUpdate()` - автоматическое тестирование добавления и удаления реакций
- `testReaction(chatId, messageId, reactionTypeId, userId, eventType)` - ручное тестирование с пользовательскими данными

## Использование

### Автоматическое обновление
Реакции обновляются автоматически при получении соответствующих событий через WebSocket.

### Тестирование
Для тестирования функциональности:

1. Откройте консоль браузера на странице с чатом
2. Выполните скрипт `debug-centrifuge-fix.js`
3. Используйте функции из `window.debugCentrifuge`:

```javascript
// Автоматический тест
window.debugCentrifuge.testReactionUpdate();

// Ручной тест
window.debugCentrifuge.testReaction(16, 338, 3, "user-id", "new_reaction");

// Тест с пользовательскими данными
window.debugCentrifuge.testWithData({
  event_type: "new_reaction",
  data: {
    chat_id: 16,
    message_id: 338,
    reaction_type_id: 3,
    user_id: "test-user"
  }
});
```

## Преимущества

1. **Мгновенное обновление** - реакции обновляются без задержки
2. **Оптимизация производительности** - локальное обновление без перезагрузки всех сообщений
3. **Надежность** - fallback на полную перезагрузку при ошибках
4. **Совместимость** - поддержка различных форматов данных от сервера

## Отладка

Для отладки включено подробное логирование:
- `🎭 Обработка реакции:` - получение данных реакции
- `🎭 Данные реакции:` - извлеченные данные
- `➕ Добавлена реакция локально:` - успешное добавление
- `➖ Удалена реакция локально:` - успешное удаление
- `✅ Реакция обновлена локально для сообщения:` - успешное обновление
- `❌ Ошибка при локальном обновлении реакции:` - ошибка обновления

## Поддерживаемые форматы данных реакций

Композабл `useReactions` поддерживает следующие форматы:

1. Стандартный формат с массивом пользователей:
```javascript
{
  reaction_type_id: 3,
  users: [{user: "user-id", user_name: "User Name"}]
}
```

2. Прямой формат с user_id:
```javascript
{
  user_id: "user-id",
  reaction_type_id: 3
}
```

3. Формат с полем user:
```javascript
{
  user: "user-id",
  reaction_type: 3
}
```
