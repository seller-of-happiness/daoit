# Оптимизация поиска пользователей в чатах

## Проблема
В компоненте `InviteUsersDialog.vue` поиск пользователей для добавления в группу происходил через запросы на сервер, хотя данные всех сотрудников уже доступны в локальном хранилище `apiStore.employees`.

## Решение
Заменили серверные запросы на локальный поиск по данным из `apiStore.employees`.

## Основные изменения

### 1. Локальный поиск (`searchLocalEmployees`)
- Поиск по имени, email, должности и отделению
- Поддержка частичного совпадения (substring match)
- Ограничение результатов до 50 записей для производительности
- Преобразование данных из формата `IEmployee` (apiStore) в формат `IEmployee` (chat)

### 2. Улучшенная обработка ошибок
- Предварительная загрузка данных сотрудников при открытии диалога
- Fallback на серверный поиск при ошибках локального поиска
- Автоматическая загрузка данных при их отсутствии

### 3. Улучшенный UX
- Уменьшено время debounce с 300ms до 100ms (локальный поиск быстрее)
- Добавлен счетчик найденных результатов
- Улучшен placeholder с подсказкой о возможностях поиска
- Добавлено отображение email сотрудников

### 4. Оптимизированная фильтрация
- Исключение пользователей, уже состоящих в чате
- Поддержка разных форматов ID участников (user/user_uuid)

## Преимущества

1. **Производительность**: Локальный поиск работает мгновенно
2. **Снижение нагрузки на сервер**: Нет запросов при каждом вводе
3. **Offline-готовность**: Поиск работает даже при проблемах с сетью
4. **Лучший UX**: Быстрые результаты и больше информации о пользователях
