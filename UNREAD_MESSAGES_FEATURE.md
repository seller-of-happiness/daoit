# Функция непрочитанных сообщений в заголовке

## Описание

Добавлена функция отображения количества непрочитанных сообщений в заголовке браузера, которая работает только когда вкладка с чатом неактивна.

## Как это работает

### Основной функционал

1. **Отслеживание непрочитанных сообщений**: Когда приходит новое сообщение от другого пользователя и вкладка браузера неактивна, счетчик увеличивается на 1.

2. **Обновление заголовка**: Заголовок браузера меняется с `DAO-MED` на `(3) DAO-MED` (где 3 - количество непрочитанных сообщений).

3. **Автоматический сброс**: Счетчик автоматически сбрасывается в следующих случаях:
   - Пользователь переключается на вкладку с чатом
   - Пользователь открывает любой чат
   - Пользователь выходит из системы

### Технические детали

#### Composable `useUnreadMessages`

Создан composable `/src/refactoring/modules/chat/composables/useUnreadMessages.ts` с следующим API:

```typescript
export function useUnreadMessages() {
  return {
    unreadCount: Ref<number>,        // Текущее количество непрочитанных
    incrementUnread: () => void,     // Увеличить счетчик на 1
    resetUnread: () => void,         // Сбросить счетчик в 0
    cleanup: () => void,             // Очистка слушателей событий
    isTabActive: Readonly<Ref<boolean>> // Активна ли вкладка
  }
}
```

#### Интеграция в чат-систему

Функционал интегрирован в `chatStore.ts`:

1. **При получении нового сообщения** (`handleNewMessage`): Если сообщение от другого пользователя, вызывается `incrementUnread()`
2. **При открытии чата** (`openChat`): Счетчик сбрасывается
3. **При сбросе состояния** (`resetInitialization`): Счетчик сбрасывается

#### Отслеживание активности вкладки

Используется браузерный API `document.visibilitychange` для отслеживания:
- Когда вкладка становится активной - счетчик автоматически сбрасывается
- Счетчик увеличивается только если вкладка неактивна

## Использование

### Автоматическое использование

Функционал работает автоматически для всех пользователей чата. Никаких дополнительных действий не требуется.

### Программное использование

Если нужно получить доступ к счетчику в компонентах:

```typescript
import { useChatStore } from '@/refactoring/modules/chat/stores/chatStore'

const chatStore = useChatStore()

// Получить количество непрочитанных сообщений в заголовке
const titleUnreadCount = chatStore.titleUnreadCount
```

### Прямое использование composable

```typescript
import { useUnreadMessages } from '@/refactoring/modules/chat/composables/useUnreadMessages'

const { unreadCount, incrementUnread, resetUnread, cleanup } = useUnreadMessages()

// Увеличить счетчик
incrementUnread()

// Сбросить счетчик
resetUnread()

// Очистить при размонтировании компонента
onUnmounted(() => {
  cleanup()
})
```

## Примеры работы

### Сценарий 1: Пользователь работает в другой вкладке
1. Пользователь открыл чат в одной вкладке
2. Переключился на другую вкладку (например, работает в документах)
3. Приходит новое сообщение в чат
4. Заголовок вкладки с чатом меняется на `(1) DAO-MED`
5. Приходит еще одно сообщение - заголовок становится `(2) DAO-MED`

### Сценарий 2: Пользователь возвращается к чату
1. Пользователь видит в заголовке `(3) DAO-MED`
2. Переключается на вкладку с чатом
3. Заголовок автоматически возвращается к `DAO-MED`
4. Счетчик сброшен

### Сценарий 3: Пользователь активно использует чат
1. Пользователь находится на вкладке с чатом
2. Приходят новые сообщения
3. Заголовок остается `DAO-MED` (счетчик не увеличивается)
4. Уведомления работают как обычно (звук, визуальные индикаторы)
