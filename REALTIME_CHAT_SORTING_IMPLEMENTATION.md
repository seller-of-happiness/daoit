# Реализация реального времени сортировки чатов

## Проблема
Чаты в списке сортировались только при обновлении страницы. При получении новых сообщений или реакций через WebSocket чаты не поднимались наверх списка в реальном времени.

## Решение

### 1. Обновлена функция `handleNewMessage`
- Теперь обновляет поля `last_message` и `last_message_id` для всех чатов (активных и неактивных)
- Вызывает `sortChatsByLastMessage()` после получения каждого нового сообщения
- Чат с новым сообщением автоматически поднимается наверх списка

### 2. Обновлена функция `handleReactionUpdate`
- Добавлено поле `last_activity_time` для отслеживания времени последней активности (реакций)
- При получении реакции чат поднимается наверх списка
- Реакции теперь считаются активностью чата

### 3. Обновлена функция сортировки `compareChatsByLastMessage`
- Учитывает новое поле `last_activity_time` с наивысшим приоритетом
- Порядок приоритетов: время последней активности > время последнего сообщения > время создания чата

### 4. Обновлен интерфейс `IChat`
- Добавлено поле `last_activity_time?: string` для отслеживания времени последней активности

### 5. Проверены все WebSocket обработчики
- Убедились что во всех необходимых местах вызывается `sortChatsByLastMessage()`
- Исправлены TypeScript ошибки в обработчиках ошибок

## Результат
Теперь при получении:
- Нового сообщения в любой чат
- Новой реакции на сообщение
- Любой другой активности связанной с чатом

Соответствующий чат **немедленно** поднимается на первое место в списке чатов без необходимости обновления страницы.
